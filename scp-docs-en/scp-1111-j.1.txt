21 Nov 2012 20:53

Doping imputities make each circuit unique on a molecular level, and it's not even uncommon for impurities to cause a circuit to malfunction. But that circuit will malfunction the same way every time. A given electrical current passes through a particular medium in a certain way, it can't change it's path and pattern anymore than the sun can set in the west.

Dopant migration is one of those things that's technically true, but doesn't mean what it seems to. The details are very boring, but suffice to say that recreating data from dopant migration is like recreating all the e-mails you've ever written by examing the wear patterns on your fingerprints from the keyboard. Theoretically, it's possible, but mind-blowingly unlikely.

I get what you're saying about the programming errors though. Point of view matters. I just don't think that way about it because I spend so much time working at the machine code level. The programmer is just some other user that messed with my machine before I got to it.

Like I said up-thread, I think I'm just the wrong audience for this.

27 Nov 2012 15:45

The problem with intermittent problems is that they are, by nature, intermittent. It may be difficult or impossible to recreate *reliably*. Case in point: the first time I had a power supply die on me. My computer would, at seemingly random intervals, shut off. Sometimes it would be after being on for hours, sometimes Windows wasn't even loaded yet; sometimes it would be during incredibly heavy loads, sometimes it was when it was sitting idle with nothing running but background stuff. It turned out that there had been a small bit of once we found that, if I had been capable (or willing) to run it with the power supply open and exposed, I could have recreated it every time, but there was no way to actually tell before hand.

THAT, is where the seemingly random, intermittent problems most usually come from. Not a part that's *failed*, but one that is just on the very edge of complete failure.

From where I sit, which is in a network operations center, where my job is to watch for alerts for all sorts of software, networking, and hardware errors from datacenters full of servers, computers act up for many, many reasons, most of which can be abstracted to, operating environments are dynamic, not static. The hard drive that was 84% full is now 85% full. Something changed the boot order since the last reboot, so that one application tried to pull a config file down from another server before the network connection was up. That Web server is responding slowly, because an application is indexing the documents and using a lot of disk I/O.

I was making a joke above about system logs. Look at the system logs for any system that's apparently running smoothly, and see how many warnings and error messages there are. Operating systems are complex and messy, and sometimes processes conflict with each other or run out of storage space.
